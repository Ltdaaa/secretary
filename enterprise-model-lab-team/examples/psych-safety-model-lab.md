# Example: Psychological Prompt Safety AI Company

## Input

Build a Fortune-500-style AI technology company team that designs, trains, evaluates, launches, operates, and commercializes a psychological prompt safety model product.

## Assumptions

- Demo must run without GPU.
- Existing local assets include psychological prompt datasets, a Skill Guard safety engine, and THETA model documentation.
- Real model training is optional for the first hackathon demo; the core demo must show a complete company workflow and reproducible delivery package.

## Team Execution Snapshot

| --- | --- | --- |

| Product Manager Agent | Define the psychological prompt safety product workflow | PRD and acceptance criteria |

| Fine-tuning Engineer Agent | Fine-tune only if baseline fails target recall or precision | Fine-tune decision memo |
| Evaluation & Test Agent | Evaluate accuracy, high-risk recall, false positive rate, latency, and red-team robustness | Evaluation protocol |
| Safety & Compliance Agent | Review self-harm, therapy-disruption, privacy, and misuse risks | Risk register |
| Agent Engineer Agent | Package the workflow as a reusable Skill plus CLI generator | Skill and script commands |
| Brand & Demo Agent | Show one input becoming a full AI company delivery package | Brand demo playbook |

## Recommended Demo Flow

1. User enters: "Design a model lab workflow for psychological prompt safety."
2. The Skill shows the Fortune-500-style AI company roster.
3. The CLI generates the delivery workspace.
4. The demo opens `project-brief.yaml`, `delivery-board.md`, and `03-model-selection.md`.
5. The presenter explains: "This is a digital AI Lab. It does data, model, training, fine-tuning, testing, safety, and product delivery as a repeatable operating system."

## Model Selection Starter Matrix

| Route | Strength | Weakness | Use First? |
| --- | --- | --- | --- |
| Rule + TF-IDF classifier | Fast, local, cheap, explainable | Limited semantic generalization | Yes |
| Prompt-only frontier model | Strong reasoning, no training needed | Cost, privacy, API dependency | Demo comparison |
| Embedding + classifier | Better semantic grouping | Requires embedding model setup | Second baseline |
| LoRA fine-tuned open model | Strong domain adaptation | Needs GPU and stronger eval discipline | Later extension |

## Champion Message

This project wins attention by making AI product development look like an operating company: each executive and specialist function has a job, every handoff creates evidence, and the final result is a reproducible company-level AI product delivery package rather than a one-off chat answer.
