项目判断

GPO 不是文本/LLM 项目，而是 SAM + DINOv2/feature matching + RL/DQN prompt optimizer 的计算机视觉分割项目。

正确路线不是直接 LLM fine-tune，而是把它产品化为 SAM Prompt Optimization Layer。

职能发现建议
产品经理核心价值是优化 SAM 提示点/框，提高分割效果。定位为面向医学图像、VOC-like、few-shot 分割的 prompt optimization layer。
数据治理存在图像、bbox、feature、prompt indices，但缺少统一 manifest。新增 `dataset_manifest.csv/jsonl`，统一记录图像、mask、特征、prompt、split。
模型分析已有 SAM、DINOv2、RL/DQN、feature matching 资产。保留 SAM 冻结，用 RL prompt optimizer 做自适应策略层。
评测测试存在 FSS、VOC、segmentation 评测脚本。统一输出 mIoU、Dice、boundary F1、prompt count、latency、reward curve。
MLOps有 best/final 模型文件，但缺少模型注册表。建立模型 registry，绑定 config、checkpoint、训练日志和评测结果。

推荐模型路线

  1. 保持 SAM 冻结。
  2. 使用 DINOv2/feature matching 提供视觉表征。
  3. 使用现有 RL/DQN prompt optimizer 作为自适应策略层。
  4. 统一 FSS、ISIC/Kvasir、VOC 等任务评测指标。
  5. 产品化为可交互的 segmentation prompt optimization demo。
