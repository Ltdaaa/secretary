## What The Project Appears To Do

GPO proposes a plug-and-play prompt optimizer for Segment Anything Model. Its key idea is to improve point/box prompts through a reinforcement-learning policy over feature and spatial relationships, so SAM can produce better masks without retraining SAM itself.

The repository includes:

- `main_FSS.py`, `main_ISIC.py`, `main_VOC.py`: dataset-specific segmentation/optimization entry points.
- `main_DQN.py`, `train_DQN.py`: reinforcement learning components.
- `evaluate_FSS.py`, `evaluate_VOC.py`, `evaluate_segmentation.py`: evaluation entry points.
- `feature_matching/`: DINOv2-style feature matching assets.
- `segmenter/`: Segment Anything related code.
- `train_data/`: prompt, bbox, and feature artifacts.
- `train/*/episode_results/training_log.txt`: training evidence.
- `model/` and `train/*/best_models`: saved policy/model artifacts.

## Model Route Recommendation

Primary route:

1. Treat GPO as a **computer-vision prompt optimization product**, not a text model product.
2. Keep SAM frozen.
3. Use DINOv2/feature matching for representation.
4. Use the trained RL point/box optimizer as the adaptive policy.
5. Evaluate on FSS, ISIC/Kvasir, and VOC-like datasets with segmentation metrics.

Do not start with:

- LLM fine-tuning.
- Generic CSV classifier training.
- Rewriting the full segmentation stack.

## Productization Plan

1. **Dataset manifest:** generate one table that maps image, mask, bbox, feature file, prompt file, dataset, split, and task.
2. **Unified evaluation:** wrap `evaluate_FSS.py`, `evaluate_VOC.py`, and `evaluate_segmentation.py` into a common output schema.
3. **Model registry:** record `best_model.pt`, `node_best_model.pkl`, `box_best_model.pkl`, config, reward stats, and training log path.
4. **Demo service:** expose an input image/reference mask/initial prompt flow that shows before/after segmentation.
5. **Monitoring:** track reward distribution, segmentation metric drift, prompt count, and runtime latency.
