# 企业级大模型训练设计团队参赛执行方案

## 作品定位

项目名称：Enterprise Model Lab Team

一句话定位：一支模拟世界 500 强科技公司运作方式的多 Agent AI 公司团队，能够从业务目标出发，自动完成市场机会、产品定义、用户研究、数据搜查、数据治理、模型分析、训练方案、微调方案、评测测试、MLOps、法务合规、财务资源、运营增长、客户成功和交付展示。

比赛目标不是展示一个普通聊天助手，而是展示一个可复用的 AI 数字组织。评委应在 3 分钟内看懂团队结构，在 5 分钟内看到端到端交付结果，在 10 分钟内相信它能迁移到真实企业研发流程。

## 冠军级判断标准

1. 业务闭环完整：从一句话需求到项目交付包，不停留在单点问答。
2. 分工像企业：每个 Agent 有职责、输入、输出、升级条件和质量门禁。
3. 结果可验证：每个阶段都有文档、指标、检查清单或可运行脚本。
4. 技术可信：包含模型基线、数据版本、训练/微调策略、评测集、消融实验和风险控制。
5. 演示直接：展示网站必须有一键示例、团队组织图、流水线图、输出样例和效率对比。
6. 可复用：能服务不同任务，不是只为一个演示写死。

## 端到端流程

1. 输入需求：用户提供行业、目标、用户群体、已有数据、算力约束、交付期限。
2. 指挥官拆解：生成项目章程、成功指标、风险列表和角色任务。
3. 数据搜查：收集公开数据源、内部数据字段、数据许可和可用性。
4. 数据治理：生成数据卡、清洗规则、标注规范、训练/验证/测试划分。
5. 模型分析：比较闭源 API、开源基座、小模型、本地模型和成本。
6. 训练设计：给出训练路线、实验矩阵、超参、算力预算和失败回滚方案。
7. 微调设计：给出 SFT/LoRA/DPO 等路线，明确何时不该微调。
8. 测试评测：构建离线指标、人工评测、红队集、上线阈值和回归测试。
9. Skill 封装：把流程变成可调用的 Agent 团队 Skill。
10. 展示交付：输出 Demo、案例、网站材料、评委讲稿和代码说明。

## 首个可交付版本

当前阶段先交付可用 Skill，不直接训练真实大模型。原因是比赛初期最关键的是展示可复用的多 Agent 公司级组织能力，真实训练可以作为后续增强模块接入 THETA 和现有心理安全模型资产。

首版包含：

- `enterprise-model-lab-team/SKILL.md`：核心 Skill。
- `enterprise-model-lab-team/scripts/create_workspace.py`：一键生成项目工作区。
- `enterprise-model-lab-team/scripts/company_os.py`：真实执行器，用于扫描项目资产、分析数据、推荐模型路线、计算准备度并生成运行报告。
- `enterprise-model-lab-team/templates/project-brief.yaml`：项目输入模板。
- `enterprise-model-lab-team/templates/delivery-board.md`：交付看板模板。
- `enterprise-model-lab-team/templates/evaluation-rubric.md`：评审评分模板。

## 冲奖路线图

### 第 1 天：完成可运行 Skill

- Skill 能被 Codex/OpenClaw 读取。
- 脚本能生成标准项目目录。
- 执行器能扫描真实项目并生成 `run-report.md`、`asset-inventory.json`、`readiness-score.json`。
- 用 1 个案例跑通从需求到交付模板的闭环。

### 第 2 天：接入真实案例

推荐案例：心理安全大模型训练设计团队。

理由：当前仓库已有心理提示词检测、THETA、模型评测和论文材料，可直接转化为真实技术资产，不需要从零虚构。

### 第 3 天：做展示网站

展示首页必须包含：

- 一句话价值主张。
- 企业级组织架构图。
- 一键示例输入。
- 多 Agent 流水线。
- 输出样例。
- 评测和安全门禁。
- 与传统团队的效率对比。

### 第 4 天：补齐技术可信度

- 给出模型选型矩阵。
- 给出数据卡。
- 给出训练/微调实验矩阵。
- 给出评测协议。
- 给出失败处理和上线门禁。

### 第 5 天：打磨冠军叙事

评审讲稿应围绕三句话：

1. 这不是聊天机器人，而是一支 AI 研发组织。
2. 它把真实 AI Lab 的数据、模型、训练、微调、测试和安全流程自动化。
3. 它已经能一键生成企业级交付包，并可接入真实训练项目继续扩展。

## 稳定性原则

- 不依赖外部 API 才能启动。
- 不要求 GPU 才能演示。
- 所有输出先以 Markdown/YAML 固化，方便评委审查。
- 模型训练作为增强能力，不能阻塞核心 Demo。
- 不声称完成真实训练，除非有训练日志、数据版本和评测结果。
